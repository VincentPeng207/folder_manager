# 文件夹管理器应用实现指南

## 项目概述

文件夹管理器是一款轻量级桌面应用程序，允许用户创建、保存和管理文件夹结构模板，并能够根据这些模板在文件系统中生成标准化的文件夹层次结构。本指南将帮助您按照五个阶段逐步实现这个应用程序。
采用C/C++语言，使用Win32 API开发。
编译使用mingw64。

## 关键功能

1. 创建和编辑文件夹结构模板
2. 保存、加载和删除模板
3. 从现有目录导入文件夹结构
4. 基于模板生成文件夹结构
5. 支持文件夹名称前缀（日期、路径或自定义）

## 第二步：实现创建模板、保存模板的功能

### 目标
实现模板的基本管理功能，包括创建、编辑、保存和加载模板。

### 核心功能

1. **模板创建功能**
   - **新建模板**：
     - 点击"新建模板"按钮时，清空当前树视图，创建新的空模板对象
     - 系统自动分配唯一ID（使用UUID生成）
     - 默认命名为"新模板"
   - **目录节点添加**：
     - 用户点击"创建目录"按钮添加新节点
     - 位置确定逻辑：
       - 如果当前有选中节点，新节点作为其子节点
       - 如果没有选中节点，新节点创建在根级别
     - 自动命名为"新建目录"，如果已存在则自动编号（如"新建目录 2"）
     - 创建后立即进入编辑状态允许用户修改名称

2. **模板编辑功能**
   - **重命名文件夹**：
     - 允许用户点击选中的节点名称进入编辑状态
     - 实时验证名称合法性（禁止使用Windows文件系统非法字符：`< > : " / \ | ? *`）
     - 同级目录不允许重名（不区分大小写）
   - **删除文件夹**：
     - 用户选中节点后按Delete键删除
     - 删除父节点时，所有子节点一并删除
     - 删除前提供确认提示
   - **移动文件夹（忽略，不需要实现）**：
     - 支持通过拖放操作重新排列节点
     - 确保不会创建循环引用（不能将节点拖放到其子节点下）

3. **模板存储功能**
   - **保存模板**：
     - 保存位置：用户AppData目录下的专用文件夹
     - 存储格式：JSON格式，包含模板元数据和完整节点结构
     - 验证过程：保存前验证所有目录名称的合法性
   - **加载模板**：
     - 从存储位置加载可用模板列表显示在下拉菜单中
     - 用户选择模板时加载并在树视图中显示
   - **删除模板**：
     - 用户点击"删除模板"按钮移除选定模板
     - 未保存的模板不需要删除操作

        }

### 用户交互流程

1. **创建新模板**：
   - 用户点击"新建模板"按钮
   - 系统清空当前显示，创建新的空模板
   - 树视图显示为空

2. **添加目录节点**：
   - 用户点击"创建目录"按钮
   - 系统在当前选中位置添加新节点
   - 用户输入有效的目录名称
  
3. **删除目录节点**：
   - 用户选中节点后按Delete键删除
   - 删除父节点时，所有子节点一并删除
   - 删除前提供确认提示

4. **保存模板**：
   - 用户点击"保存模板"按钮
   - 如果是新模板，系统弹出对话框让用户输入模板名称
   - 系统验证并保存模板到AppData目录
   - 保存成功后显示确认消息

5. **加载模板**：
   - 用户从顶部下拉菜单选择已保存的模板
   - 系统加载模板并在树视图中显示其结构

## 第三步：实现导入目录功能

### 目标
实现从现有文件系统导入目录结构到模板的功能，允许用户快速导入已有的文件夹结构。

### 核心功能

1. **导入接口设计**
   - 添加"导入目录"按钮，点击后打开文件夹选择对话框
   - 用户选择要导入的根文件夹
   - 系统扫描该文件夹及其子文件夹结构，创建对应的节点树

2. **目录扫描功能**
   - **深度优先扫描**：从根目录开始，递归扫描所有子目录
   - **性能考虑**：
     - 设置最大扫描深度（默认10层）以防止过深的目录结构
     - 每层最多扫描一定数量的目录（默认10个）以避免性能问题
     - 总数量限制为100个目录
     - 使用后台线程进行扫描，避免UI卡顿
   - **进度反馈**：为长时间扫描提供进度指示器

3. **导入位置逻辑**
   - **位置确定规则**：
     - 如果当前有选中节点：导入的结构作为该节点的子结构
     - 如果没有选中节点：导入的结构添加到根级别
   - **节点创建**：为每个扫描到的目录创建对应的节点，保持原有层次结构

4. **冲突处理**
   - 检测导入结构与现有结构的潜在冲突
   - 如存在同名节点，提供选项：覆盖、重命名或跳过
   - 保持节点ID的唯一性，为新导入的每个节点分配新ID

### 用户交互流程

1. **启动导入**：
   - 用户选择目标节点（或不选择以导入到根级别）
   - 用户点击"导入目录"按钮
   - 系统打开文件夹选择对话框

2. **选择目录**：
   - 用户浏览并选择要导入的文件夹
   - 用户确认选择

3. **处理导入**：
   - 系统显示进度对话框，开始扫描目录结构
   - 对于大型目录结构，提供取消选项

4. **完成导入**：
   - 扫描完成后，系统在目录树视图中显示导入的结构
   - 显示导入摘要（如导入的目录数量）
   - 提示用户保存模板以保留导入的结构

## 第四步：实现在用户电脑目录中根据模板创建目录功能

### 目标
实现基于当前模板在用户选定位置生成实际文件夹结构的功能。

### 核心功能

1. **生成接口设计**
   - 添加"生成目录结构"按钮，点击后启动生成流程
   - 实现生成选项对话框，提供各种生成配置

2. **目标位置选择**
   - 打开系统文件夹选择对话框，让用户指定生成位置
   - 验证所选位置的可写性和有效性

3. **生成模式选择**
   - **直接生成模式**：在选定位置直接创建模板中定义的文件夹结构
   - **创建顶层文件夹模式**：在选定位置先创建一个根文件夹，然后在其中创建模板结构
   - 如选择创建顶层文件夹，提供输入字段让用户指定顶层文件夹名称

4. **文件夹创建功能**
   - 实现递归创建文件夹的功能
   - 处理长路径问题（Windows路径长度限制为260字符）
   - 实现错误处理和异常恢复机制
   - 使用后台线程执行创建操作，避免UI卡顿

### 生成过程逻辑

1. **准备阶段**
   - 验证模板有效性（至少包含一个节点）
   - 验证目标位置可写性
   - 构建完整的节点路径映射（计算每个节点的完整路径）

2. **创建阶段**
   - 按层次顺序（从浅到深）创建文件夹
   - 对于每个节点：
     - 计算完整路径
     - 创建对应的文件夹
     - 处理可能的创建失败（权限不足、名称冲突等）

3. **完成阶段**
   - 提供操作摘要（创建的文件夹数量）
   - 提供选项打开生成的根目录

### 用户交互流程

1. **启动生成**：
   - 用户点击"生成目录结构"按钮
   - 系统检查当前模板有效性

2. **配置生成选项**：
   - 系统显示生成选项对话框
   - 用户选择是否创建顶层文件夹
   - 如选择创建顶层文件夹，用户输入顶层文件夹名称

3. **选择目标位置**：
   - 系统打开文件夹选择对话框
   - 用户浏览并选择目标位置
   - 用户确认选择

4. **执行创建**：
   - 系统显示进度对话框，开始创建文件夹
   - （忽略）对于大型结构，提供取消选项

5. **完成创建**：
   - 创建完成后，显示成功消息
   - 提供选项打开生成的文件夹

## 第五步：实现添加前缀功能

### 目标
实现为生成的文件夹名称添加前缀的功能，支持日期、路径和自定义前缀。

### 核心功能

1. **前缀设置界面**
   - 在生成选项对话框中添加前缀设置部分
   - 包含启用/禁用前缀的复选框
   - 提供前缀类型选择（单选按钮组）：日期前缀、路径前缀、自定义前缀
   - 前缀和目录名之间用_拼接

2. **日期前缀功能**
   - 实现多种日期格式选项：
     - YYYY-MM-DD：如"2023-05-20_"
     - YYYYMMDD：如"20230520_"
     - YYYY_MM_DD：如"2023_05_20_"
   - 使用当前系统日期生成前缀

3. **路径前缀功能**
   - 模板内所有父文件夹(包括用户创建的顶层文件夹)名称以_拼接生成前缀，前缀不包括目标目录。
   - 例子：目标目录为D:\Projects，目录模板为"\a\b\c",那么目录c的前缀为"a_b_"，c的目录名变为'a_b_c'，c的完整路径为D:\Projects\a\a_b\a_b_c。
   - 例子2：目标目录为D:\Projects，目录模板为"\1\2\3",用户如果创建了顶层文件夹e，那么目录3的前缀为"e_1_2_"，3的目录名变为'e_1_2_3'，3的完整路径为D:\Projects\e_1\e_1_2\e_1_2_3。
   - 提供选项包含或排除路径分隔符（忽略）

4. **自定义前缀功能**
   - 提供输入字段让用户指定自定义前缀文本
   - 验证前缀文本合法性（不含文件名非法字符）
   - 路径为前缀_目录名
   - （忽略）自动在前缀后添加下划线分隔符（如用户未添加）

5. **前缀应用逻辑**
   - 前缀应用到每个生成的文件夹名称
   - 仅在实际创建时应用前缀，不修改原模板
   - 验证添加前缀后的名称长度和合法性

### 前缀验证规则

1. **合法性验证**
   - 前缀不能包含文件名非法字符：`< > : " / \ | ? *`
   - 避免以点(.)或下划线(_)开头或结尾
   - 自动处理前缀与文件名之间的分隔符（下划线）

2. **长度限制**
   - 前缀最大长度：50字符
   - 确保添加前缀后的完整文件夹名称不超过255字符

### 用户交互流程

1. **配置前缀**：
   - 用户在生成选项对话框中勾选"目录名称添加前缀"
   - 用户选择前缀类型（日期、路径或自定义）
   - 根据选择配置相应选项（日期格式或自定义文本）

2. **预览前缀**：
   - （忽略）系统实时显示前缀预览示例
   - 对于日期前缀，显示当前日期的格式化结果
   - 对于路径前缀，基于当前所选目标路径显示预览
   - 对于自定义前缀，显示用户输入的文本

3. **应用前缀**：
   - 用户确认生成选项后，系统在创建文件夹时应用设定的前缀
   - 系统验证每个添加前缀后的名称
   - 如出现无效名称，提供错误提示
