# Folder Manager

简体中文 | [English](README.md)

Folder Manager 是一个 Windows 目录结构模板生成器，用于快速创建重复项目文件夹。

它最初是我给自己用的工具：当一个项目、客户资料、合同资料或其他工作流需要反复创建相同的文件夹结构时，可以先维护一个模板，然后一键生成对应目录。

## 功能

- 创建和编辑目录结构模板
- 保存、加载和删除模板
- 从已有文件夹导入目录结构
- 按模板生成实际文件夹
- 生成时可选择创建顶层文件夹
- 支持目录名前缀：
  - 日期前缀
  - 路径前缀
  - 自定义前缀
- 支持中文和英文界面
- 支持通过 Windows 资源管理器右键菜单在当前位置打开工具

## 适用场景

- 重复创建项目目录
- 统一资料归档结构
- 为客户、案件、合同、设计稿、视频素材等建立固定文件夹层级
- 从已有目录反向整理成可复用模板

## 平台

- Windows
- C++17
- MinGW-w64
- ImGui + GLFW

## 构建

需要先安装 MinGW-w64，并确保 `g++`、`windres` 和 `mingw32-make` 已在 `PATH` 中。

```powershell
mingw32-make
```

构建完成后，可执行文件位于：

```text
build/bin/FolderManagerImGui.exe
```

也可以直接运行：

```powershell
mingw32-make run
```

清理构建产物：

```powershell
mingw32-make clean
```

## 使用

1. 启动程序。
2. 点击 `+ 模板` 创建新模板。
3. 点击 `+ 目录` 添加目录节点。
4. 通过双击或右键菜单重命名目录。
5. 保存模板。
6. 点击生成按钮，选择目标位置并生成目录结构。

没有已保存模板时，程序会显示一个中性的示例模板，方便新用户理解操作流程。这个示例不会写入模板存储，除非你手动保存。

模板会保存在当前用户的 AppData 目录下：

```text
%APPDATA%\FolderManager\Templates
```

语言偏好会保存在：

```text
%APPDATA%\FolderManager\settings.ini
```

首次启动时会根据系统语言选择中文或英文，也可以通过菜单栏的 `语言` 手动切换。

## 右键菜单

项目里保留了右键菜单注册相关文件，用于把工具挂到 Windows 资源管理器目录右键菜单中。

如果使用安装包，可以在安装时选择是否添加右键菜单。未安装右键菜单时，仍然可以直接运行程序并手动选择目标目录。
